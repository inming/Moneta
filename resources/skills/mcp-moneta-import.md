# Moneta 账单导入助手

## 何时使用

当用户想要从本地文件（Excel/CSV/银行账单/支付宝/微信账单）导入交易记录到 Moneta 记账软件时使用本技能。

## 前置检查

- 确认 Moneta 应用正在运行
- 确认 MCP Server 已配置（在 Moneta 设置页点击「配置 Claude Desktop MCP」）

## 工作流程

### Step 1: 获取元数据

先调用工具获取当前分类和操作人：

```typescript
// 获取所有启用的分类（含 AI 描述）
const categories = await get_categories()

// 获取操作人列表
const operators = await get_operators()
```

### Step 2: 分析文件

用户会提供：
- 文件路径（如 `/Users/xxx/Downloads/alipay_2024.csv`）
- 或直接粘贴文件内容

你需要：

1. **识别日期列**：统一转换为 `YYYY-MM-DD` 格式
2. **识别金额列**：
   - 处理正负号、千分号、货币符号
   - ✅ 金额支持**正数和负数**，负数表示退款/冲正
   - ❌ 金额不能为 0
3. **识别描述/商品名称列**：用于匹配分类
4. **判断交易类型**（收入/支出/投资）：映射为 `expense`/`income`/`investment`

### Step 3: 智能分类匹配

结合分类的 `name` 和 `description` 进行语义匹配：

| 账单关键词 | 建议分类 | 匹配依据（description） |
|-----------|---------|----------------------|
| 美团、饿了么、外卖、堂食、餐厅 | 正餐 | "外卖、堂食、食堂、餐厅" |
| 滴滴、uber、地铁、公交、加油、停车费 | 交通 | "地铁、公交、打车、加油、停车费" |
| 淘宝、京东、超市、日用品、清洁用品 | 百货 | "超市购物、日用品、清洁用品" |
| Steam、电影票、KTV、游戏、演出 | 娱乐 | "游戏、电影、演出、KTV" |
| 房租、房贷、月供 | 住房 | "房租、房贷月供" |
| 水费、电费、燃气费 | 水电燃气 | - |
| 零食、饮料、甜品 | 零食 | - |
| 手机费、宽带费、话费 | 通讯 | - |
| 软件、订阅、会员、工具 | 效率 | "软件订阅、工具 App、会员服务" |
| 工资、薪资、月薪 | 工资 | "固定薪资收入"（收入分类） |
| 奖金、年终奖、绩效 | 奖金 | "年终奖、绩效奖金等"（收入分类） |
| 理财、利息、收益 | 理财 | "投资收益、利息等"（收入分类） |
| 红包、礼金 | 红包 | "收到的红包、礼金"（收入分类） |
| 基金定投、申购 | 基金 | "基金定投、申购等"（投资分类） |
| 股票买入 | 股票 | "股票买入"（投资分类） |

**不确定时留空 `category_id`**，不要在确认界面让用户选择。

### Step 4: 发送数据

调用 `send_transactions()`：

```json
{
  "transactions": [
    {
      "date": "2024-01-15",
      "type": "expense",
      "amount": 128.50,
      "category_id": 5,
      "category_name": "正餐",
      "description": "美团外卖 - 午餐",
      "operator_id": 1,
      "operator_name": "本人"
    },
    {
      "date": "2024-01-16",
      "type": "expense",
      "amount": -35.00,
      "category_id": 5,
      "category_name": "正餐",
      "description": "美团外卖 - 退款",
      "operator_id": 1,
      "operator_name": "本人"
    }
  ],
  "source": "支付宝账单 2024年1月"
}
```

Moneta 会自动打开确认界面，用户完成最终编辑后入库。

## 金额处理规则

不同账单源的金额格式不同：

| 账单源格式 | 处理方式 | 示例 |
|-----------|----------|------|
| **正负表示收支** | 保留正负值，type 根据上下文判断 | -128.50 → amount: -128.50 |
| **单独收/支列** | 金额取正值，type 映射 expense/income | 支出 128.50 → amount: 128.50, type: expense |
| **文字标识** | 提取数字，根据关键词判断正负 | "退款 50元" → amount: -50 |

## 注意事项

- ✅ 金额支持**正数和负数**，负数表示退款/冲正场景
- ✅ 如果源数据用正负号表示收支（如银行账单），直接保留原值
- ✅ 如果源数据用单独的「收入/支出」列标识，金额取正数，用 type 字段区分
- ❌ 金额不能为 0（Moneta 会拒绝）
- 日期严格使用 `YYYY-MM-DD` 格式
- 复杂文件（如多 sheet、非常规格式）先询问用户确认字段映射
- 不强行匹配不确定的分类
