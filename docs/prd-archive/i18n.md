# 多语言支持 — 功能规格归档
> 从 PRD.md Section 3.9 归档。此功能已实现（v0.6）。

### 3.9 多语言

#### 3.9.1 功能概述

Moneta 支持多语言界面，使应用能够服务不同语言的用户。用户可在设置页切换界面语言，应用立即生效无需重启。

#### 3.9.2 支持的语言

**初期（v0.6）**：
- 简体中文（zh-CN）
- 英文（en-US）

**未来扩展**：
- 繁体中文（zh-TW）
- 日语（ja-JP）
- 韩语（ko-KR）

#### 3.9.3 语言切换

**位置**：设置页新增「语言」Tab，位于「MCP 配置」和「安全设置」之间

**界面设计**：
- 使用 Radio.Group 单选组件
- 选项：「简体中文」和「English」
- 点击「保存」按钮后生效
- 保存后界面立即切换，无需重启应用

**配置存储**：
- 保存到用户本地配置文件（`app.getPath('userData')/config.json`）
- 新增 `language` 字段，值为 `zh-CN` 或 `en-US`

#### 3.9.4 默认语言检测

**首次启动时**：
- 自动检测系统语言（`app.getLocale()` 或 `navigator.language`）
- 系统语言为 `zh-*` 时，默认使用简体中文
- 其他情况默认使用英文
- 用户可随时在设置页切换

**后续启动时**：
- 读取配置文件中的 `language` 字段
- 无配置时回退到系统语言检测

#### 3.9.5 翻译范围

**需要翻译的内容**：
- 所有 UI 文本（按钮、标签、菜单项、占位符、标题）
- 消息提示（成功/错误/警告 Toast）
- Modal 对话框内容
- 表格列标题
- 表单验证消息
- 日期时间格式文本
- 交易类型的显示名称（`expense` → "消费"/"Expense"）

**不翻译的内容**（用户数据保持原始输入）：
- 分类名称（用户创建时使用的语言）
- 分类的 AI 描述字段
- 操作人姓名
- 交易描述
- Ant Design 组件自带文本（通过 `ConfigProvider` locale 自动切换）
- dayjs 时间文本（通过 `dayjs.locale()` 自动切换）

**设计原则**：
- 用户数据（分类、操作人、描述）是用户创建的内容，不应被系统翻译
- 中文用户可以创建中文分类，英文用户可以创建英文分类
- 仅 UI 元素根据用户选择的语言动态切换

#### 3.9.6 系统预置分类的处理

**初期方案（v0.6）**：
- 系统预置分类（如"正餐"、"交通"等）保持中文，不提供翻译
- 英文用户首次使用时，可以停用中文预置分类，创建自己语言的分类

**未来增强（v0.7+）**：
- 为系统预置分类增加多语言翻译表
- 数据库增加 `name_zh`、`name_en` 字段
- 根据界面语言显示对应翻译

#### 3.9.7 技术实现

**国际化框架**：
- 使用 i18next + react-i18next
- 按模块组织翻译文件（命名空间）
- 支持变量插值（如「成功删除 {{count}} 条记录」）

**翻译文件组织**：
```
src/renderer/src/locales/
├── zh-CN/
│   ├── common.json       # 通用文本
│   ├── navigation.json   # 侧栏导航
│   ├── transactions.json # 数据浏览页
│   ├── statistics.json   # 统计报表页
│   ├── settings.json     # 设置页
│   ├── auth.json         # 安全认证
│   ├── ai.json           # AI 识别
│   ├── import.json       # 导入确认
│   └── messages.json     # 消息提示
└── en-US/
    └── (同上结构)
```

**Ant Design 和 dayjs 联动**：
- 切换语言时同步更新 Ant Design 的 locale（通过 `ConfigProvider`）
- 同步更新 dayjs 的 locale（通过 `dayjs.locale()`）

#### 3.9.8 验收标准

| # | 验收条件 | 说明 |
|---|---------|------|
| AC-I18N-1 | 设置页新增「语言」Tab | 位于「MCP 配置」和「安全设置」之间 |
| AC-I18N-2 | 支持切换中文/英文 | 使用 Radio.Group 单选 |
| AC-I18N-3 | 点击保存后立即生效 | 无需重启应用，界面立即切换语言 |
| AC-I18N-4 | 配置持久化 | 关闭应用后重启，语言保持用户选择 |
| AC-I18N-5 | 首次启动检测系统语言 | 中文系统默认中文，其他系统默认英文 |
| AC-I18N-6 | 所有页面文本翻译完整 | 包括导航、表格、按钮、表单、消息提示 |
| AC-I18N-7 | Ant Design 组件联动 | 切换语言后分页、空状态等组件文本自动翻译 |
| AC-I18N-8 | dayjs 日期格式联动 | 切换语言后日期格式自动调整 |
| AC-I18N-9 | 消息提示支持变量插值 | 如「成功删除 5 条记录」正确显示变量 |
| AC-I18N-10 | 翻译文本符合目标语言习惯 | 英文翻译自然流畅，无机翻痕迹 |
| AC-I18N-11 | 用户数据保持原始输入 | 切换语言后，分类名称、操作人、描述等用户数据不变 |
| AC-I18N-12 | 错误提示完整翻译 | 包括表单验证、API 错误等所有错误消息 |
| AC-I18N-13 | 交易类型显示正确翻译 | 数据浏览页、统计页等处的"消费/收入/投资"根据语言显示 Expense/Income/Investment |
| AC-I18N-14 | 中文用户创建的分类不被翻译 | 英文界面下仍显示中文分类名（如"正餐"） |
