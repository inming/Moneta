# AI 图片识别 — 功能规格归档
> 从 PRD.md Section 3.2 归档。此功能已实现。

### 3.2 AI 图片识别

#### 3.2.1 使用场景

用户拍摄或截取以下页面的截图/照片：
- 支付宝/微信支付账单页面
- 电商平台（淘宝、京东等）订单列表
- 银行 App 交易明细
- 纸质小票、发票
- 其他任何包含交易信息的页面或凭证

#### 3.2.2 识别流程

```
┌──────────────┐    ┌───────────┐    ┌──────────────┐    ┌──────────┐    ┌────────┐
│ 上传/粘贴     │ →  │ AI 提取    │ →  │ 智能匹配分类  │ →  │ 用户确认  │ →  │ 批量入库│
│ 一张或多张图片 │    │ 多条交易   │    │ 未识别留空    │    │ 补充修改  │    │        │
└──────────────┘    └───────────┘    └──────────────┘    └──────────┘    └────────┘
```

#### 3.2.3 图片输入

- **输入方式**：支持以下两种方式提交图片：
  - 文件上传：点击上传区域或拖拽图片文件
  - 剪贴板粘贴：通过 Ctrl+V / Cmd+V 粘贴截图
- **多图支持**：一次可提交多张图片，所有图片在**单次 API 请求**中发送给 AI 一并识别
- **格式支持**：JPEG、PNG、WebP、BMP
- **大小限制**：单张图片不超过 10 MB，单次请求总量不超过 20 MB
- **图片预览**：上传后展示缩略图列表，支持删除某张图片后再发起识别

#### 3.2.4 AI 识别与分类匹配

1. **AI 提取**：调用已配置的 AI 模型（参见 §3.7 AI 模型配置），从图片中提取结构化交易数据：
   - 金额
   - 描述/商品名
   - 交易类型（消费/收入/投资）
   - 注：日期不需要 AI 提取，统一使用用户在识别页选定的「记账日期」
2. **智能分类匹配**：AI 返回结果后，系统根据每条交易的描述信息自动匹配最可能的分类：
   - 已知模式匹配（如"美团外卖"→ 正餐，"滴滴出行"→ 交通，"Steam"→ 娱乐）
   - 可利用 AI 在识别时直接建议分类（将现有分类列表作为上下文传入 prompt）
   - AI prompt 中以 JSON 数组格式传入分类列表，每项包含 `name`（分类名）和可选的 `description`（AI 描述），如：`[{"name":"住房","description":"房租、房贷月供"},{"name":"保险"}]`，避免特殊字符导致解析歧义
   - 无法识别的分类字段**留空并高亮标记**，提示用户必须手动选择
3. **分类完整性约束**：每条交易必须有分类才能提交，未填写分类的条目阻止入库

#### 3.2.5 确认界面

- 以可编辑表格形式展示所有提取的条目
- 可编辑字段：类型、金额、分类、描述、操作人
- 日期不在表格列中，由页面顶部的「记账日期」选择器统一指定，所有条目共享同一日期。**日期选择器初始为空**，用户必须主动选择日期后才能提交
- **分类为空的行高亮警示**（红色背景色），引导用户补充
- **类型颜色标记**：不同交易类型的行使用不同背景色（消费=淡橙色、收入=淡绿色、投资=淡蓝色），与系统 Tag 颜色体系一致
- 用户可逐条删除不需要的条目
- **手工插入条目**：支持在任意位置插入新的空行，方便补充 AI 未识别到的交易
  - 每行操作列提供「插入」按钮，点击后在该行**上方**插入一条空行
  - 新插入的行字段默认与当前行相同（类型、操作人），金额为空、分类为空、描述为空
  - 插入后新行自动进入编辑状态（或保持与其他行一致的编辑模式）
  - 表格底部提供「添加一行」按钮，用于在末尾追加新行
- **操作人联动**：操作人选择器**初始为空**，用户必须主动选择操作人后才能提交；选择后所有行自动同步为该操作人，后续切换操作人时所有行同步更新
- 顶部显示识别摘要：共识别 N 条，待补充分类 M 条
- **提交校验**：点击「确认录入」时检查以下必填项，任一不满足则提示用户并阻止提交：
  - 分类：所有条目的分类字段不为空
  - 日期：记账日期已选择（AI 识别为全局日期；MCP 导入为每行日期）
  - 操作人：全局操作人已选择

#### 3.2.6 批量入库

- 用户确认所有信息无误后，点击「确认录入」批量写入数据库
- 入库结果反馈：成功录入 N 条交易记录

#### 3.2.7 验收标准

| # | 验收条件 | 说明 |
|---|---------|------|
| AC-1 | 支持上传至少 5 张图片并一次性识别 | 测试多图同时提交 |
| AC-2 | 支持粘贴剪贴板图片 | Ctrl+V / Cmd+V 粘贴截图后出现在图片列表中 |
| AC-3 | AI 返回结构化数据包含金额、描述、类型 | 验证 JSON 解析正确 |
| AC-4 | 已知描述自动匹配到正确分类 | 如"美团外卖"匹配"正餐" |
| AC-5 | 未识别分类的条目高亮显示 | 分类列为空时行背景/边框有视觉提示 |
| AC-6 | 所有条目分类不为空时才能提交 | 存在空分类时点击提交弹出错误提示 |
| AC-7 | 确认界面中每个字段可编辑 | 用户可修改类型、金额、分类、描述、操作人 |
| AC-8 | 批量入库后数据正确写入数据库 | 查看交易列表验证数据一致 |
| AC-9 | 无可用 AI 模型时给出明确提示 | 未配置任何 API Key 时引导用户到设置页 |
| AC-10 | AI 请求失败时展示错误信息并允许重试 | 网络超时、API 报错等场景 |
| AC-11 | 不同交易类型用不同背景色标记 | 消费=淡橙色、收入=淡绿色、投资=淡蓝色 |
| AC-12 | 切换操作人后所有待插入行自动更新 | 识别完成后切换操作人，所有行同步更新 |
| AC-13 | 识别过程中可查看实时日志 | 点击"查看日志"链接可随时查看 AI 调用日志 |
| AC-14 | 支持在任意位置插入新行 | 点击「插入」按钮可在当前行上方添加空行 |
| AC-15 | 插入行后该行自动进入编辑状态 | 便于用户立即填写数据 |
| AC-16 | 新插入行继承当前行的类型和操作人 | 金额、分类、描述为空，减少重复选择 |
| AC-17 | 支持在表格末尾追加新行 | 通过「添加一行」按钮在底部添加 |
| AC-18 | 插入的行参与提交校验 | 分类为空时同样阻止提交并高亮提示 |
| AC-19 | 导入确认页日期选择器初始为空 | AI 识别：全局日期选择器无默认值；MCP 导入：每行日期字段清空（即使 MCP 数据携带日期） |
| AC-20 | 导入确认页操作人选择器初始为空 | 全局操作人选择器无默认值，不自动选中第一个操作人 |
| AC-21 | 日期或操作人未设置时阻止提交 | 点击「确认录入」时若日期或操作人未选择，弹出提示并阻止提交 |
| AC-22 | MCP 导入清空 AI 提供的日期 | 即使 MCP 数据中包含日期，进入确认页后也清空，要求用户手动确认 |
