#### 3.3.4 分类可配置

- 用户可**新增**自定义分类（如「宁宁」「萌萌」等按家庭成员命名的分类）
- 用户可**编辑**分类名称
- 用户可**删除**不需要的分类（已关联交易的分类标记为停用，不物理删除）
- 用户可**排序**分类的显示顺序
- 消费、收入和投资分类独立管理
- 每个分类可设置 **AI 描述**（选填），用于说明该分类的典型消费场景或关键词示例，辅助 AI 图片识别时进行更准确的分类匹配
- AI 描述仅供 AI 识别时使用（传入 prompt 作为上下文），不在交易列表等其他业务界面显示
- 默认分类可预置 AI 描述（见 §3.3.1 ~ §3.3.3 各分类表中的「AI 描述」列），用户可自由修改
- AI 描述长度限制 100 字符

#### 3.3.5 分类描述验收标准

| # | 验收条件 | 说明 |
|---|---------|------|
| AC-C1 | 分类数据结构包含 AI 描述字段 | DB categories 表新增 description 列，类型 TEXT，可为空 |
| AC-C2 | 设置页分类管理可编辑 AI 描述 | 新增/编辑分类弹窗中可填写 AI 描述 |
| AC-C3 | AI 描述为选填字段 | 不填不影响分类正常使用和 AI 识别 |
| AC-C4 | AI 描述长度限制 100 字符 | 超出时前端提示 |
| AC-C5 | 默认分类预置 AI 描述 | 新安装时默认分类带有预设描述（见 §3.3.1 ~ §3.3.3） |
| AC-C6 | AI prompt 包含分类描述 | 分类以 JSON 数组传入 prompt，含 name 和可选 description 字段 |
| AC-C7 | 已有用户升级时自动迁移 | 已有分类的 description 默认为空，不影响现有数据 |
