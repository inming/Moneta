# Moneta — 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品愿景

Moneta 是一款面向个人和家庭的桌面端记账软件，帮助用户轻松记录日常收支、分类管理财务、可视化分析消费习惯。通过 AI 截图识别能力，大幅降低手动录入的成本。

### 1.2 目标用户

- 有记账习惯的个人用户（从 Excel 记账迁移而来）
- 家庭用户（多人共同记账，按操作人区分）

### 1.3 核心价值

| 价值点 | 描述 |
|--------|------|
| 高效录入 | AI 截图识别 + 批量录入，告别逐条手动输入 |
| 数据沉淀 | SQLite 本地存储，数据完全属于用户 |
| 直观分析 | 对标 Excel 透视表的交叉统计 + 可视化图表 |
| 平滑迁移 | 兼容用户现有 Excel 数据格式，无缝导入历史数据 |

## 2. 核心概念

| 概念 | 定义 |
|------|------|
| **交易（Transaction）** | 一条收支或投资记录，是系统的最小数据单元 |
| **类型（Type）** | 交易的性质：消费、收入 或 投资。投资用于区分非消费性支出（如基金、股票等） |
| **分类（Category）** | 交易的归类标签，消费、收入和投资各有独立分类体系 |
| **操作人（Operator）** | 记录这笔账的家庭成员，简单文本标识 |
| **记账日期** | 每次开始记账时选定的日期，本次会话所有新增条目默认使用该日期 |

### 3.1 交易数据结构

对齐用户现有 Excel 表结构：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 日期 | Date | ✅ | 默认为当前「记账日期」，可单条修改 |
| 添加时间 | DateTime | 系统生成 | 记录该条交易写入系统的时间，精确到秒，不可手动编辑 |
| 类型 | Enum | ✅ | "消费"、"收入" 或 "投资" |
| 金额 | Number | ✅ | 可为正数或负数，精确到分（两位小数） |
| 分类 | String | ✅ | 从分类列表中选择 |
| 描述 | String | ❌ | 自由文本备注 |
| 操作人 | String | ❌ | 从已有操作人列表选择 |

## 4. 非功能性需求

| 需求 | 目标 |
|------|------|
| **离线优先** | 所有核心功能离线可用（AI 截图识别除外，需联网调用 API） |
| **数据安全** | SQLite 数据库文件存储在用户本地，不上传至任何云端；数据库使用 SQLCipher 加密（v0.7+）；应用启动时 PIN 码锁屏保护 |
| **启动性能** | 冷启动时间 < 3 秒 |
| **大数据量** | 流畅支持 10 万条以上交易记录 |
| **数据备份** | 用户可手动导出数据库文件进行备份 |
| **平台支持** | Windows 10+ 和 macOS 12+ |
| **多语言支持** | 支持中文和英文界面，首次启动时自动检测系统语言 |

## 5. 迭代路线图

> 已实现版本的详细任务清单见 [iteration-history.md](prd-archive/iteration-history.md)。

### 未来考虑

#### 近期计划

- **预算管理**：按分类设置月度/年度预算额度，超支预警提示，从"记录"到"管控"的关键一步
- **周期性交易 / 录入自动分摊**：支持周期性交易模板（如房租、工资、话费等固定项），减少重复录入；支持一次性大额支出自动分摊到多个月（如一次性交一年房租，自动拆分为 12 条月度记录）
- **数据备份**：支持手动/自动备份 SQLite 数据库到用户指定目录，防止数据丢失

#### 远期考虑

- 多币种支持
- 标签系统（一条交易可打多个标签）
