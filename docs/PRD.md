# Moneta — 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品愿景

Moneta 是一款面向个人和家庭的桌面端记账软件，帮助用户轻松记录日常收支、分类管理财务、可视化分析消费习惯。通过 AI 截图识别能力，大幅降低手动录入的成本。

### 1.2 目标用户

- 有记账习惯的个人用户（从 Excel 记账迁移而来）
- 家庭用户（多人共同记账，按操作人区分）

### 1.3 核心价值

| 价值点 | 描述 |
|--------|------|
| 高效录入 | AI 截图识别 + 批量录入，告别逐条手动输入 |
| 数据沉淀 | SQLite 本地存储，数据完全属于用户 |
| 直观分析 | 对标 Excel 透视表的交叉统计 + 可视化图表 |
| 平滑迁移 | 兼容用户现有 Excel 数据格式，无缝导入历史数据 |

---

## 2. 核心概念

| 概念 | 定义 |
|------|------|
| **交易（Transaction）** | 一条收入或支出记录，是系统的最小数据单元 |
| **类型（Type）** | 交易的方向：消费 或 收入 |
| **分类（Category）** | 交易的归类标签，消费和收入各有独立分类体系 |
| **操作人（Operator）** | 记录这笔账的家庭成员，简单文本标识 |
| **记账日期** | 每次开始记账时选定的日期，本次会话所有新增条目默认使用该日期 |

## 3. 功能模块

### 3.1 交易管理

#### 3.1.1 交易数据结构

对齐用户现有 Excel 表结构：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 日期 | Date | ✅ | 默认为当前「记账日期」，可单条修改 |
| 类型 | Enum | ✅ | "消费" 或 "收入" |
| 金额 | Number | ✅ | 正数，精确到分（两位小数） |
| 分类 | String | ✅ | 从分类列表中选择 |
| 描述 | String | ❌ | 自由文本备注 |
| 操作人 | String | ❌ | 从已有操作人列表选择或输入新名称 |

#### 3.1.2 录入模式

1. **选定记账日期**：每次开始记账时，用户先选择一个日期（默认今天），本次会话中所有新增条目自动采用该日期
2. **快速录入表单**：选择类型 → 输入金额 → 选择分类 → （可选）填写描述和操作人 → 提交
3. **连续录入**：提交后表单自动清空（保留类型、日期、操作人），方便连续录入多条
4. **编辑与删除**：支持对已有交易的修改和删除

#### 3.1.3 交易列表

- 默认按日期倒序展示
- 支持按日期范围、类型、分类、操作人筛选
- 支持关键词搜索（搜索描述字段）

### 3.2 AI 截图识别

#### 3.2.1 使用场景

用户拍摄或截取以下页面的截图：
- 支付宝/微信支付账单页面
- 电商平台（淘宝、京东等）订单列表
- 银行 App 交易明细
- 其他任何包含交易信息的页面

#### 3.2.2 识别流程

```
┌──────────┐    ┌───────────┐    ┌──────────────┐    ┌──────────┐    ┌────────┐
│ 粘贴/上传 │ →  │ AI 提取    │ →  │ 自动匹配分类  │ →  │ 用户确认  │ →  │ 批量入库│
│ 截图      │    │ 多条交易   │    │ 未识别留空    │    │ 补充信息  │    │        │
└──────────┘    └───────────┘    └──────────────┘    └──────────┘    └────────┘
```

#### 3.2.3 详细说明

1. **输入**：用户通过粘贴（Ctrl+V）或文件上传提供截图
2. **AI 提取**：调用 AI 视觉 API，从截图中提取结构化交易数据：
   - 日期（如截图中有）
   - 金额
   - 描述/商品名
   - 交易类型（消费/收入）
3. **分类匹配**：根据提取到的描述信息，自动匹配最可能的分类：
   - 已知模式匹配（如"餐"→ 正餐，"地铁"→ 交通）
   - 无法识别的分类字段**留空**，等待用户手动选择
4. **确认界面**：以可编辑表格形式展示所有提取的条目：
   - 每个字段均可修改
   - 分类为空的条目高亮提示用户补充
   - 用户可删除不需要的条目
   - 日期默认使用当前「记账日期」（如 AI 提取到日期则优先使用）
5. **入库**：用户确认所有信息无误后，点击「确认录入」批量写入数据库

### 3.3 分类管理

#### 3.3.1 默认消费分类

基于用户现有数据预置以下分类：

| 分类 | 分类 | 分类 | 分类 |
|------|------|------|------|
| 住房 | 保险 | 水电燃气 | 正餐 |
| 食材 | 零食 | 交通 | 百货 |
| 娱乐 | 电子数码 | 鞋服 | 通讯 |
| 效率 | 物业 | 教育 | 社交 |
| 宠物 | 医疗 | 家具 | 电器 |
| 工作 | 其他 | | |

#### 3.3.2 默认收入分类

| 分类 | 说明 |
|------|------|
| 工资 | 固定薪资收入 |
| 奖金 | 年终奖、绩效奖金等 |
| 理财 | 投资收益、利息等 |
| 红包 | 收到的红包、礼金 |
| 副业 | 兼职、稿费等 |
| 报销 | 费用报销 |
| 其他 | 未归类的收入 |

#### 3.3.3 分类可配置

- 用户可**新增**自定义分类（如「宁宁」「萌萌」等按家庭成员命名的分类）
- 用户可**编辑**分类名称
- 用户可**删除**不需要的分类（已关联交易的分类标记为停用，不物理删除）
- 用户可**排序**分类的显示顺序
- 消费分类和收入分类独立管理

### 3.4 统计报表

#### 3.4.1 交叉统计表（核心视图）

对标用户现有 Excel 透视表：

- **行维度**：分类（Category）
- **列维度**：时间，支持三级层次：
  - 月度（1月、2月…12月）
  - 季度汇总（第一季 ~ 第四季）
  - 年度汇总
- **值**：金额合计（SUM）
- **筛选器**：
  - 按类型筛选：消费 / 收入
  - 按年份切换
  - 按操作人筛选
- 底部显示**总计行**

#### 3.4.2 辅助图表

| 图表类型 | 用途 | 说明 |
|----------|------|------|
| 饼图 | 分类占比 | 展示选定时间段内各分类的金额占比 |
| 折线图 | 月度趋势 | 展示收支随时间的变化趋势 |
| 柱状图 | 月度对比 | 收入 vs 消费的月度对比 |

#### 3.4.3 快捷统计

- 首页仪表盘展示：本月收支汇总、本月 vs 上月对比、本年累计

### 3.5 数据导入/导出

#### 3.5.1 导入

- **支持格式**：Excel（.xlsx）、CSV（.csv）
- **兼容现有格式**：自动识别用户现有 Excel 的列结构（日期、类型、金额、分组、描述、操作人）
- **导入流程**：
  1. 选择文件
  2. 预览解析结果（展示前 N 行）
  3. 列映射确认（如列名不完全匹配，允许手动映射）
  4. 分类匹配（自动匹配已有分类，未知分类提示用户处理：创建新分类 / 映射到已有分类）
  5. 确认导入
- **去重策略**：基于日期+金额+描述检测重复，提示用户选择跳过或覆盖

#### 3.5.2 导出

- **支持格式**：Excel（.xlsx）、CSV（.csv）
- **导出范围**：支持按时间范围、类型、分类筛选后导出
- **导出列**：与用户现有 Excel 格式一致（日期、类型、金额、分组、描述、操作人）

### 3.6 操作人

- 录入时可从下拉列表选择已有操作人，或直接输入新名称
- 新输入的名称自动保存到操作人列表
- 统计和交易列表可按操作人筛选
- 不涉及任何用户认证、登录功能

## 4. 非功能性需求

| 需求 | 目标 |
|------|------|
| **离线优先** | 所有核心功能离线可用（AI 截图识别除外，需联网调用 API） |
| **数据安全** | SQLite 数据库文件存储在用户本地，不上传至任何云端 |
| **启动性能** | 冷启动时间 < 3 秒 |
| **大数据量** | 流畅支持 10 万条以上交易记录 |
| **数据备份** | 用户可手动导出数据库文件进行备份 |
| **平台支持** | Windows 10+ 和 macOS 12+ |
