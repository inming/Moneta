# Moneta — 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品愿景

Moneta 是一款面向个人和家庭的桌面端记账软件，帮助用户轻松记录日常收支、分类管理财务、可视化分析消费习惯。通过 AI 截图识别能力，大幅降低手动录入的成本。

### 1.2 目标用户

- 有记账习惯的个人用户（从 Excel 记账迁移而来）
- 家庭用户（多人共同记账，按操作人区分）

### 1.3 核心价值

| 价值点 | 描述 |
|--------|------|
| 高效录入 | AI 截图识别 + 批量录入，告别逐条手动输入 |
| 数据沉淀 | SQLite 本地存储，数据完全属于用户 |
| 直观分析 | 对标 Excel 透视表的交叉统计 + 可视化图表 |
| 平滑迁移 | 兼容用户现有 Excel 数据格式，无缝导入历史数据 |

---

## 2. 核心概念

| 概念 | 定义 |
|------|------|
| **交易（Transaction）** | 一条收支或投资记录，是系统的最小数据单元 |
| **类型（Type）** | 交易的性质：消费、收入 或 投资。投资用于区分非消费性支出（如基金、股票等） |
| **分类（Category）** | 交易的归类标签，消费、收入和投资各有独立分类体系 |
| **操作人（Operator）** | 记录这笔账的家庭成员，简单文本标识 |
| **记账日期** | 每次开始记账时选定的日期，本次会话所有新增条目默认使用该日期 |

## 3. 功能模块

### 3.1 交易管理

#### 3.1.1 交易数据结构

对齐用户现有 Excel 表结构：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 日期 | Date | ✅ | 默认为当前「记账日期」，可单条修改 |
| 添加时间 | DateTime | 系统生成 | 记录该条交易写入系统的时间，精确到秒，用于排序和审计，不可手动编辑 |
| 类型 | Enum | ✅ | "消费"、"收入" 或 "投资" |
| 金额 | Number | ✅ | 可为正数或负数，精确到分（两位小数）；负数用于退款、冲正等场景 |
| 分类 | String | ✅ | 从分类列表中选择 |
| 描述 | String | ❌ | 自由文本备注 |
| 操作人 | String | ❌ | 从已有操作人列表选择或输入新名称 |

#### 3.1.2 录入模式

1. **选定记账日期**：每次开始记账时，用户先选择一个日期（默认今天），本次会话中所有新增条目自动采用该日期
2. **快速录入表单**：选择类型 → 输入金额 → 选择分类 → （可选）填写描述和操作人 → 提交
3. **连续录入**：提交后表单自动清空（保留类型、日期、操作人），方便连续录入多条
4. **编辑与删除**：支持对已有交易的修改和删除

- 金额支持录入负数，用于退款、冲正等需要抵减原交易的场景

#### 3.1.3 交易列表

- 默认按日期倒序展示
- 支持按日期范围、类型、分类、操作人筛选
- 支持关键词搜索（搜索描述字段）
- 支持按添加时间排序，便于查看最新录入的数据
- 数据浏览页仅展示交易记录，分类和操作人管理已移至设置页（§3.3、§3.6）

#### 3.1.4 数据浏览与编辑

数据浏览页以单一表格形式展示所有交易记录（不含分类和操作人 Tab，这些在设置页管理）。

##### 表格结构

- 列：日期、添加时间、类型、金额、分类、描述、操作人、操作（编辑/删除按钮）
- 分页展示，支持切换每页条数（20/50/100）
- 表头显示总记录数
- 添加时间列为只读字段，显示系统实际写入该条记录的时间

##### 行内编辑

- 点击编辑按钮或双击行进入编辑状态，支持修改：
  - **日期**：日期选择器
  - **类型**：下拉选择（消费/收入/投资）
  - **金额**：数字输入框
  - **分类**：下拉选择（根据当前类型动态加载对应分类列表）
  - **描述**：文本输入框
  - **操作人**：下拉选择
- **添加时间**不可编辑，编辑已有记录时保持原值不变
- 类型变更后，若原分类不属于新类型，分类自动清空
- 编辑后保存即时写入数据库

##### 删除

- 每行操作列提供删除按钮
- 删除前弹出确认提示（防止误操作）
- 支持批量选择 + 批量删除

##### 新增一行

- 表格上方提供「新增」按钮
- 新增行各字段默认为空，但**日期字段记忆上次用户输入的值**（方便连续录入同一天的数据）
- 日期记忆仅当前会话有效，应用关闭后清空
- 新增行需填写必填字段（日期、类型、金额、分类）后方可保存

##### 按列排序

- 点击列头按该列升序/降序排序
- 支持排序的列：日期、添加时间、类型、金额、分类、操作人
- 添加时间排序基于该条记录的实际写入时间，而非记账日期
- 排序作用于全量数据结果，而非仅当前页前端重排

##### 验收标准

|---|---------|------|
| AC-D1 | 数据浏览页仅展示交易记录表格 | 不含分类和操作人 Tab |
| AC-D2 | 支持行内编辑所有字段 | 日期、类型、金额、分类、描述、操作人均可编辑 |
| AC-D3 | 类型变更后分类自动清空 | 切换类型后若分类不匹配则清空 |
| AC-D4 | 编辑即时保存到数据库 | 保存后刷新页面数据一致 |
| AC-D5 | 支持单行删除（有确认提示） | 防止误删 |
| AC-D6 | 支持批量选择和批量删除 | 勾选多行后一键删除 |
| AC-D7 | 支持新增一行 | 点击「新增」按钮添加空行 |
| AC-D8 | 新增行日期记忆上次输入值 | 连续添加时自动填充上次日期 |

### 3.2 AI 图片识别

#### 3.2.1 使用场景

用户拍摄或截取以下页面的截图/照片：
- 支付宝/微信支付账单页面
- 电商平台（淘宝、京东等）订单列表
- 银行 App 交易明细
- 纸质小票、发票
- 其他任何包含交易信息的页面或凭证

#### 3.2.2 识别流程

```
┌──────────────┐    ┌───────────┐    ┌──────────────┐    ┌──────────┐    ┌────────┐
│ 上传/粘贴     │ →  │ AI 提取    │ →  │ 智能匹配分类  │ →  │ 用户确认  │ →  │ 批量入库│
│ 一张或多张图片 │    │ 多条交易   │    │ 未识别留空    │    │ 补充修改  │    │        │
└──────────────┘    └───────────┘    └──────────────┘    └──────────┘    └────────┘
```

#### 3.2.3 图片输入

- **输入方式**：支持以下两种方式提交图片：
  - 文件上传：点击上传区域或拖拽图片文件
  - 剪贴板粘贴：通过 Ctrl+V / Cmd+V 粘贴截图
- **多图支持**：一次可提交多张图片，所有图片在**单次 API 请求**中发送给 AI 一并识别
- **格式支持**：JPEG、PNG、WebP、BMP
- **大小限制**：单张图片不超过 10 MB，单次请求总量不超过 20 MB
- **图片预览**：上传后展示缩略图列表，支持删除某张图片后再发起识别

#### 3.2.4 AI 识别与分类匹配

1. **AI 提取**：调用已配置的 AI 模型（参见 §3.7 AI 模型配置），从图片中提取结构化交易数据：
   - 金额
   - 描述/商品名
   - 交易类型（消费/收入/投资）
   - 注：日期不需要 AI 提取，统一使用用户在识别页选定的「记账日期」
2. **智能分类匹配**：AI 返回结果后，系统根据每条交易的描述信息自动匹配最可能的分类：
   - 已知模式匹配（如"美团外卖"→ 正餐，"滴滴出行"→ 交通，"Steam"→ 娱乐）
   - 可利用 AI 在识别时直接建议分类（将现有分类列表作为上下文传入 prompt）
   - AI prompt 中以 JSON 数组格式传入分类列表，每项包含 `name`（分类名）和可选的 `description`（AI 描述），如：`[{"name":"住房","description":"房租、房贷月供"},{"name":"保险"}]`，避免特殊字符导致解析歧义
   - 无法识别的分类字段**留空并高亮标记**，提示用户必须手动选择
3. **分类完整性约束**：每条交易必须有分类才能提交，未填写分类的条目阻止入库

#### 3.2.5 确认界面

- 以可编辑表格形式展示所有提取的条目
- 可编辑字段：类型、金额、分类、描述、操作人
- 日期不在表格列中，由页面顶部的「记账日期」选择器统一指定，所有条目共享同一日期
- **分类为空的行高亮警示**（红色背景色），引导用户补充
- **类型颜色标记**：不同交易类型的行使用不同背景色（消费=淡橙色、收入=淡绿色、投资=淡蓝色），与系统 Tag 颜色体系一致
- 用户可逐条删除不需要的条目
- **手工插入条目**：支持在任意位置插入新的空行，方便补充 AI 未识别到的交易
  - 每行操作列提供「插入」按钮，点击后在该行**上方**插入一条空行
  - 新插入的行字段默认与当前行相同（类型、操作人），金额为空、分类为空、描述为空
  - 插入后新行自动进入编辑状态（或保持与其他行一致的编辑模式）
  - 表格底部提供「添加一行」按钮，用于在末尾追加新行
- **操作人联动**：操作人默认使用当前会话操作人；识别完成后切换操作人，所有待插入行自动更新为新选择的操作人
- 顶部显示识别摘要：共识别 N 条，待补充分类 M 条
- **提交校验**：点击「确认录入」时检查所有条目的分类字段，若存在空值则提示用户并阻止提交

#### 3.2.6 批量入库

- 用户确认所有信息无误后，点击「确认录入」批量写入数据库
- 入库结果反馈：成功录入 N 条交易记录

#### 3.2.7 验收标准

| # | 验收条件 | 说明 |
|---|---------|------|
| AC-1 | 支持上传至少 5 张图片并一次性识别 | 测试多图同时提交 |
| AC-2 | 支持粘贴剪贴板图片 | Ctrl+V / Cmd+V 粘贴截图后出现在图片列表中 |
| AC-3 | AI 返回结构化数据包含金额、描述、类型 | 验证 JSON 解析正确 |

| AC-5 | 未识别分类的条目高亮显示 | 分类列为空时行背景/边框有视觉提示 |
| AC-6 | 所有条目分类不为空时才能提交 | 存在空分类时点击提交弹出错误提示 |
| AC-7 | 确认界面中每个字段可编辑 | 用户可修改类型、金额、分类、描述、操作人 |

| AC-10 | AI 请求失败时展示错误信息并允许重试 | 网络超时、API 报错等场景 |

| AC-14 | 支持在任意位置插入新行 | 点击「插入」按钮可在当前行上方添加空行 |
| AC-15 | 插入行后该行自动进入编辑状态 | 便于用户立即填写数据 |
| AC-16 | 新插入行继承当前行的类型和操作人 | 金额、分类、描述为空，减少重复选择 |
| AC-17 | 支持在表格末尾追加新行 | 通过「添加一行」按钮在底部添加 |
| AC-18 | 插入的行参与提交校验 | 分类为空时同样阻止提交并高亮提示 |

- 默认分类可预置 AI 描述（见 §3.3.1 ~ §3.3.3 各分类表中的「AI 描述」列），用户可自由修改

### 3.4 统计报表

统计报表作为独立页面，在侧栏中新增「统计报表」入口（位于「数据浏览」和「设置」之间）。页面整体分为上下两部分：上方为筛选器 + 汇总卡片区，下方为交叉统计表和辅助图表。

#### 3.4.0 导航变更

侧栏新增一项：

| 菜单项 | 路由 | 图标 | 说明 |
|--------|------|------|------|
| 统计报表 | `/statistics` | BarChartOutlined | 交叉统计 + 图表 |

侧栏顺序调整为：数据浏览 → **统计报表** → 设置 → 锁屏（底部）。

#### 3.4.1 筛选器与汇总卡片

##### 筛选器

页面顶部提供一行筛选控件，所有筛选条件变化时实时刷新下方所有区域（交叉统计表 + 图表 + 汇总卡片）：

| 筛选项 | 控件 | 默认值 | 说明 |
|--------|------|--------|------|
| 年份 | 年份选择器 | 当前年份 | 范围：数据库中最早交易年份 ~ 当前年份；无数据时仅显示当前年份；年度汇总视图下隐藏 |
| 类型 | 单选按钮组 | 消费 | 消费 / 收入 / 投资，三选一 |

- 筛选器状态不持久化，每次进入页面恢复默认值
- 类型为单选（交叉统计表同一时间只展示一种类型的分类体系）

##### 汇总卡片

筛选器下方展示一行汇总卡片（3 张），数据与当前筛选条件联动：

| 卡片 | 内容 | 说明 |
|------|------|------|
| 本月合计 | 当前年份 + 当前月份 + 当前类型的金额总计 | 若当前年份非今年，显示该年 12 月的数据 |
| 上月合计 | 上一个月同类型金额总计 | 与本月卡片的月份联动 |
| 年度合计 | 当前年份 + 当前类型的全年金额总计 | 即交叉统计表「合计」列的总计行值 |

- 金额显示使用千分号格式（与数据浏览页一致）

- 「本月」的定义：若筛选年份 = 当前年份，则取系统当前月份；若筛选年份为历史年份，则取 12 月

### 3.5 数据导入/导出

#### 3.5.1 导入

- **支持格式**：Excel（.xlsx）
- **兼容现有格式**：读取名为 "detail" 的工作表，固定列结构（日期、类型、金额、分组、描述、操作人）
- **导入模式**：全量覆盖（清空现有交易和操作人数据后重新导入）
- **导入流程**：
  1. 选择 Excel 文件
  2. 预览解析结果（展示有效记录数、操作人列表、分类列表、异常行提示）
  3. 用户确认后执行全量覆盖导入
- **分类处理**：自动匹配已有分类，未匹配的自动创建新分类
- **操作人处理**：清空后重建，根据导入数据自动创建
- **添加时间处理**：由于导入源文件不包含该字段，导入生成的交易记录以实际导入写入时间作为添加时间

#### 3.5.2 导出

- **支持格式**：Excel（.xlsx）和 CSV（.csv），用户在导出时选择
- **导出列**：与导入格式一致，固定列结构：

| 列 | 内容 | 说明 |
|----|------|------|
| 日期 | `YYYY-MM-DD` | 交易日期 |
| 类型 | 消费 / 收入 / 投资 | 中文显示名 |
| 金额 | 数字（保留两位小数） | 负数原样导出（退款/冲正场景） |
| 分组 | 分类名称 | 列名为「分组」以兼容用户现有 Excel |
| 描述 | 自由文本 | 可为空 |
| 操作人 | 操作人名称 | 可为空 |

- **导出范围**：默认导出全部交易记录；支持按以下条件筛选后导出：
  - 日期范围（起止日期）
  - 交易类型（消费 / 收入 / 投资，可多选）
  - 分类（可多选）
  - 操作人（可多选）
  - 关键词（搜索描述字段）
- **排序**：导出数据按日期升序排列（与用户 Excel 习惯一致）
- **Excel 格式细节**：
  - 工作表名称为 `detail`（与导入格式一致，导出文件可直接作为导入源）
  - 首行为表头（日期、类型、金额、分组、描述、操作人）
- **CSV 格式细节**：
  - UTF-8 with BOM 编码（确保 Excel 打开时中文不乱码）
  - 首行为表头
  - 字段分隔符为逗号，含逗号或换行的字段用双引号包裹

##### 导出流程

1. 用户在设置页「数据管理」Tab 中点击「导出数据」按钮
2. 弹出导出配置面板，包含：
   - 格式选择：Excel / CSV（默认 Excel）
   - 筛选条件（可选，默认全部导出）：日期范围、类型、分类、操作人、关键词
   - 预览：显示符合条件的记录数
3. 点击「导出」按钮，弹出系统保存对话框
4. 默认文件名：`Moneta_导出_YYYYMMDD.xlsx`（或 `.csv`），日期为当天
5. 导出完成后提示：成功导出 N 条记录到 `文件路径`

### 3.7 AI 模型配置

#### 3.7.1 概述

AI 图片识别功能依赖外部多模态 AI API。系统内置固定的模型列表，用户选择模型并填入 API Key 即可使用。不支持用户自行添加自定义模型，新增模型通过应用版本更新提供。模型配置入口位于「设置」页面。

#### 3.7.2 内置模型列表

| 模型 | 平台 | API 格式 | Endpoint | 模型标识 |
|------|------|----------|----------|----------|
| GLM-4.5V | 智谱 AI (BigModel) | OpenAI 兼容 | `https://open.bigmodel.cn/api/paas/v4` | `glm-4.5v` |
| Doubao Seed 2.0 | 火山引擎 (ByteDance) | OpenAI 兼容 | `https://ark.cn-beijing.volces.com/api/v3` | 用户填入推理接入点 ID（如 `ep-xxx`） |

- Endpoint 默认为官方地址，用户**可修改**（如使用 SiliconFlow 等第三方平台调用同一模型）
- 模型标识默认为官方标识，用户**可修改**（如第三方平台的模型标识可能不同）
- 后续版本可扩展更多内置模型（如 Kimi、GPT-4o、Claude 等），需通过代码更新

#### 3.7.3 API Key 安全存储

- 使用 Electron `safeStorage` API 对 API Key 进行加密后存储到本地配置文件
- 设置界面中 API Key 以掩码形式显示（如 `****abcd`），仅在编辑时可查看完整值
- API Key 不会以明文形式写入任何日志或配置文件

#### 3.7.4 模型切换与默认设置

- 当内置模型列表中仅有一个模型时，自动作为默认模型，无需用户手动选择
- 当内置模型列表中有多个模型时：
  - 用户可在设置页面指定一个**默认模型**
  - 在 AI 识别页面，用户可通过下拉框**临时切换**模型（不改变默认设置）
- 未配置任何 API Key 时，AI 识别页面显示引导提示，引导用户前往设置页配置

#### 3.7.5 连通性测试

- 每个模型配置 API Key 后，提供「测试连接」按钮
- 测试逻辑：发送一个最小化的 API 请求（如发送纯文本 prompt），验证 API Key 是否有效
- 测试结果：成功（显示绿色对勾 + 模型名称）或失败（显示错误原因，如认证失败、网络不可达）

#### 3.7.6 设置页 UI

- 以卡片形式展示内置模型列表
- 每个模型卡片显示：模型名称、平台名称、配置状态（已配置/未配置）、是否为默认
- 用户操作：填写/编辑 API Key、修改 Endpoint、修改模型标识、测试连接、设为默认
- 不提供删除或新增模型的入口

#### 3.7.7 验收标准

| # | 验收条件 | 说明 |
|---|---------|------|
| AC-1 | 内置模型填入 API Key 后可成功调用 AI 识别 | 验证 GLM-4.5V |

#### 3.7.8 Prompt 预览

- 分类数据实时从数据库读取，反映用户当前的分类配置（包括自定义分类和 AI 描述）
- 提供「复制」按钮，一键复制 Prompt 到剪贴板
- Modal 中 Prompt 以等宽字体展示，便于阅读

##### 验收标准

| # | 验收条件 | 说明 |
|---|---------|------|
| AC-P1 | AI 模型 Tab 中有「查看识别 Prompt」按钮 | 按钮位置在模型卡片列表下方或上方工具区 |
| AC-P2 | 点击后弹出 Modal 显示完整 Prompt | 包含角色指令、字段定义、分类列表、输出示例 |
| AC-P3 | Prompt 中的分类列表反映数据库当前状态 | 新增/编辑/停用分类后，Prompt 内容同步更新 |
| AC-P4 | 仅包含启用状态的分类 | 已停用（`is_active=0`）的分类不出现在 Prompt 中 |
| AC-P5 | Prompt 与 AI 识别时实际发送的一致 | 复用同一 `buildPrompt()` 生成逻辑，不硬编码 |
| AC-P6 | 提供一键复制功能 | 点击「复制」后 Prompt 文本写入剪贴板，提示复制成功 |
| AC-P7 | Prompt 以等宽字体可滚动展示 | Modal 内容区可滚动，长文本不截断 |

### 3.8 应用锁屏与 PIN 码解锁

#### 3.8.1 概述

为保护用户财务数据隐私，Moneta 提供应用锁屏功能。用户设置 PIN 码后，每次启动应用默认处于锁屏状态，需输入正确的 PIN 码后才能进入主界面使用。

#### 3.8.2 首次使用

- 应用首次启动时（未设置过 PIN 码），弹出引导界面要求用户设置 PIN 码
- PIN 码为固定 **4 位数字**（0-9）
- 设置流程：输入 PIN → 再次输入确认 → 两次一致后保存
- 两次输入不一致时提示错误，要求重新输入
- PIN 码设置完成后立即生效，下次启动将进入锁屏

#### 3.8.3 锁屏界面

- 应用启动后首先展示锁屏界面，覆盖整个窗口
- 界面元素：
  - 应用名称（Moneta）和图标
  - 4 位 PIN 码输入区域（每位独立输入框，输入时显示圆点掩码）
  - 输入完成后自动验证（无需点击确认按钮）
  - 错误提示区域
- 锁屏状态下不可访问应用任何功能，不可通过快捷键或其他方式绕过

#### 3.8.4 解锁流程

1. 用户在锁屏界面输入 4 位 PIN 码
2. 输入第 4 位后自动触发验证
3. 验证成功：锁屏界面消失，进入应用主界面
4. 验证失败：
   - 输入框清空并抖动，提示「PIN 码错误，请重试」
   - 显示剩余可尝试次数

#### 3.8.5 安全机制

- **PIN 码加密存储**：复用 Electron `safeStorage` API 加密后存储到本地配置文件，与 API Key 存储方式一致
- **错误次数限制**：连续输入错误 5 次后锁定应用，需等待 30 秒后才能继续尝试
- **锁定提示**：锁定期间显示倒计时，提示剩余等待时间

#### 3.8.6 PIN 码管理

PIN 码管理入口位于「设置」页面：

- **修改 PIN 码**：需先输入当前 PIN 码验证身份，再输入新 PIN 码（两次确认）
- **PIN 码始终启用**：不提供关闭锁屏的选项，PIN 码一旦设置即持续生效

#### 3.8.7 验收标准

| # | 验收条件 | 说明 |
|---|---------|------|
| AC-L1 | 首次启动弹出 PIN 码设置引导 | 未设置 PIN 时强制引导设置 |
| AC-L2 | PIN 码固定 4 位数字 | 仅接受 0-9 数字输入 |
| AC-L3 | 设置 PIN 需两次确认一致 | 两次不一致时提示错误 |
| AC-L4 | 每次启动应用显示锁屏界面 | 主界面在解锁前不可见、不可操作 |
| AC-L5 | 输入正确 PIN 后解锁进入主界面 | 验证成功后锁屏消失 |
| AC-L6 | 输入错误 PIN 后提示并清空 | 显示错误信息和剩余次数 |
| AC-L7 | 连续 5 次错误后锁定 30 秒 | 锁定期间显示倒计时 |
| AC-L8 | PIN 码加密存储 | 使用 safeStorage API，配置文件中不含明文 |
| AC-L9 | 设置页可修改 PIN 码 | 需验证旧 PIN 后才能设置新 PIN |
| AC-L10 | 锁屏界面输入掩码显示 | 输入数字显示为圆点 |
| AC-L11 | 输入 4 位后自动验证 | 无需手动点击确认按钮 |

## 4. 非功能性需求

| 需求 | 目标 |
|------|------|
| **离线优先** | 所有核心功能离线可用（AI 截图识别除外，需联网调用 API） |
| **数据安全** | SQLite 数据库文件存储在用户本地，不上传至任何云端；应用启动时 PIN 码锁屏保护 |
| **启动性能** | 冷启动时间 < 3 秒 |
| **大数据量** | 流畅支持 10 万条以上交易记录 |
| **数据备份** | 用户可手动导出数据库文件进行备份 |
| **平台支持** | Windows 10+ 和 macOS 12+ |
